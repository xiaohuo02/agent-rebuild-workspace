# 03-面试讲法

## 一分钟版本

> 我在原项目旁边新建了一个重实现工作区，从 0 复刻智能客服 Agent（智能体）的核心链路。第一阶段没有直接接复杂数据库和真实大模型，而是先用 FastAPI、Vue 3 和 fake data（假数据）跑通 Agentic Workflow（智能体工作流）：Router（路由器）、Guardrails（安全护栏）、Planner（规划器）、Tool Selection（工具选择）、工具执行、Evidence（证据链）和最终回答。这样先保证架构可解释、可演示、可测试，再逐步替换成 Neo4j、Text2Cypher 和 GraphRAG。

## 为什么从假数据开始

> 我不是为了偷懒，而是为了先验证架构。Agent 项目最容易失败的地方不是工具多不多，而是链路是否清楚、边界是否可控、错误是否可降级。用假数据能先把路由、规划、工具返回和证据链测试清楚，后续替换真实工具时不会破坏上层结构。

## 前端怎么讲

> 前端采用 ChatGPT 风格对话工作台。普通用户看到的是自然对话体验，面试或调试时可以展开右侧 Agent 过程面板，看到路由结果、任务拆解、工具调用和证据链。这样既符合智能客服的使用习惯，也能说明系统不是简单聊天框，而是可解释的 Agent 工作流。

## 测试怎么讲

> 测试分两层。第一层是工程测试，用 pytest 验证 Router、工具结构和 Agent 流程。第二层是 LLM Eval（大模型评测），后续接 DeepEval 评估答案相关性、忠实度、上下文相关性、工具调用正确性和幻觉率。
